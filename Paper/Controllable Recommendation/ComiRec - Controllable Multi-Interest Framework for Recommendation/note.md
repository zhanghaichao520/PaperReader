# Controllable Multi-Interest Framework for Recommendation

## 一句话总结
这篇论文认为用户往往同时有多个兴趣，推荐系统应该显式建模这些兴趣，并允许系统或目标物品去选择当前该激活哪一个。

## 论文做了什么
作者针对序列推荐里“单一用户向量表达不了多兴趣”的问题，提出 ComiRec，让模型从历史行为中抽取多个兴趣表示，而不是把所有偏好压缩成一个 embedding。

## 核心方法
ComiRec 用多兴趣抽取模块学习多个用户兴趣向量，并在预测时根据目标物品或控制信号选择更相关的兴趣表示参与匹配，从而实现更灵活的兴趣控制。

## 结果与意义
实验表明多兴趣建模比单兴趣表示更适合复杂用户，ComiRec 也成为后续多兴趣推荐工作的一个代表性起点。
