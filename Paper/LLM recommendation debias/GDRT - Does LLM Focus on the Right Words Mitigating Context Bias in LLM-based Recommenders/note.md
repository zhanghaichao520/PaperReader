# 论文阅读笔记：Does LLM Focus on the Right Words? Mitigating Context Bias in LLM-based Recommenders

**会议/年份**：WWW 2026  

**核心关注点**：大语言模型（LLM）在推荐系统微调中产生的“上下文偏差”（Context Bias）问题及其解决方案。

---

这篇论文关注 LLM 推荐里的 **context bias**：经过 SFT 后，模型会过度依赖任务描述词、前缀词等辅助 token，而没有真正利用用户交互历史中的偏好信息。结果就是模型学会了“捷径”，推荐更容易被上下文模板牵着走，准确性和公平性都会受影响。

作者提出 **GDRT**，即基于 **Group Distributionally Robust Optimization** 的微调方法。具体做法是先按照目标物品与辅助 token 的语义相关性，把训练样本分成不同组；这个相关性通过“遮住用户历史，只看辅助 token 时模型给目标 token 的概率”来估计。然后训练时不再只优化整体平均损失，而是要求模型在各组上都保持较稳定表现，并对表现差的组动态增大权重。这样做会迫使模型减少对辅助 token 的依赖，把注意力重新拉回到真正反映用户兴趣的交互 token 上。




## 1. 研究动机与痛点 (Motivation)

在将大语言模型（LLM）应用于推荐系统（RS）时，通常会使用**监督微调（SFT）**来让模型适应推荐任务。也就是给模型输入“任务提示词+用户历史交互记录”，让模型输出“目标推荐商品”。

然而，作者通过特征消融归因分析（FAA）发现了一个致命的痛点——**上下文偏差 (Context Bias)**：
* **大模型学会了“走捷径”**：经过 SFT 后，LLM 会过度依赖输入中的**辅助词元 (Auxiliary Tokens)**（例如：固定的任务提示词“请推荐一部电影”，或者目标生成的局部前缀词），而严重忽略了真正包含用户个性化偏好的**用户交互词元 (User Interaction Tokens)**。
* **数据根源**：在训练数据中，辅助词元在每个样本里都会出现，它们与目标商品的共现率极高。模型为了快速降低 Loss，直接记住了这种浅层的相关性（捷径），而不是去深入理解长串的用户历史行为。
* **负面影响**：这种偏差不仅降低了推荐的准确率，还导致了严重的**不公平性（Unfairness）**。系统会不成比例地向用户推荐那些名字和提示词在字面上高度相关的商品，而不是用户真正喜欢的商品。

---

## 2. 核心方法 (Methodology)：GDRT

为了解决上述问题，作者提出了一种极其巧妙且高效的微调框架：**GDRT (Group Distributionally Robust Optimization-based Tuning)**。

该方法没有粗暴地修改模型架构，而是通过重构损失函数，运用“动态奖惩”的机制逼迫大模型去关注用户的历史记录。主要分为两个步骤：

### 步骤一：按“走捷径”的难易程度对目标词元进行分组 (Token Grouping)
算法首先要给大模型的偷懒能力“摸底”，找出哪些词是不看用户历史也能猜出来的。
1. **“蒙眼”测试**：遮挡住（Mask）用户的真实交互记录，只给模型看辅助词元（任务描述和生成前缀）。
2. **计算捷径得分**：计算在这种情况下模型生成目标词元 $y_{i,t}^{*}$ 的概率：
   $r(y_{i,t}^{*})=log P_{\theta}(y_{i,t}^{*}|x^{task},y_{i,<t}^{*})$
   概率越大，说明该词元与辅助词元相关性越强，越容易成为“捷径”。
3. **K-means 聚类**：根据这些得分，将所有的目标词元划分为 $G$ 个组。得分高的在强相关组（捷径词），得分低的在弱相关组（核心词）。

### 步骤二：组分布鲁棒优化 (Group DRO)
传统的 SFT 是计算所有词元预测误差的平均值（平均主义），这导致模型只要做对捷径题就能拿高分。GDRT 引入了 Group DRO 来打破这一点：
1. **拒绝偏科**：GDRT 的优化目标是让模型在**所有的词组**上都取得一致的良好表现。
2. **自适应重加权 (Adaptive Reweighting)**：在实际训练中，算法会实时监控每个组的损失（Loss）。对于那些表现差的组（通常是那些无法靠提示词蒙对、必须老老实实看用户历史的“弱相关组”），GDRT 会自动给它们**分配更大的权重 $Q(g)$**。
3. **闭式解保障效率**：作者通过严格的数学推导，给出了组权重的闭式解：$Q(g) \propto e^{\mathcal{L}(g)/\tau}$。这意味着 GDRT 只需要在标准 SFT 上多算一步指数加权，几乎没有任何额外的计算开销。

---

## 3. 结论与总结 (Conclusion)

* **机制本质**：GDRT 就像一个聪明的导师，当大模型遇到必须看用户历史才能答对的“硬骨头”时，如果答错了，就施加极其严厉的惩罚。这就倒逼大模型放弃依赖提示词的捷径，将注意力转移到核心的用户交互记录上。
* **实验效果**：在三个公开数据集上，GDRT 在几乎不增加训练时间的情况下，大幅提升了推荐准确率（NDCG@10 平均提升 24.29%），并显著改善了推荐的公平性。该方法作为一种 Loss 层的即插即用模块，可以轻松集成到现有的各类 LLM 推荐系统中。
