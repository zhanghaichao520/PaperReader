# FairLRM

这篇论文认为，现有用 LLM 缓解流行度偏置的方法太停留在“多样性”“去偏”这些表面词汇上，模型并没有真正理解 popularity bias 背后的语义和成因。因此作者想做的不是简单加一个去偏提示，而是增强 RecLLM 对“流行度偏置”这一概念本身的语义理解。

为此，论文提出 **FairLRM**。它先把流行度偏置拆成 **item-side** 和 **user-side** 两个部分，再用 **structured instruction-based prompts** 把全局物品流行度分布、用户个体偏好倾向等信息显式写进提示中，引导模型同时理解“哪些物品过热”和“这个用户是否本来就偏好小众内容”。也就是说，FairLRM 的主要做法是通过结构化提示把偏置的因果线索和双侧语义喂给模型，让模型在生成推荐时不只看表面热门信号，而能更有意识地平衡公平性与相关性。
