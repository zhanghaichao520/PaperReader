# FUDLR

这篇论文关注 LLM 推荐系统中的公平性问题，但和很多工作不同，它强调两个现实约束：一是偏置类型很多，不应每种偏置单独设计一个方法；二是大模型重训练太贵，因此去偏最好不要依赖完整再训练。

作者提出 **FUDLR**，把去偏改写成一个高效的 **machine unlearning** 问题，分两步做。第一步是 **bias identification**：通过一个可学习的 mask，为训练样本估计“它是否在诱发偏置”，优化目标同时兼顾公平性提升、准确率保持和稀疏选择。第二步是 **fast debiasing via unlearning**：根据这些被识别出的偏置样本，估计它们对模型参数的影响，并做一次近似的参数修正，把这部分影响从模型里移除。它的关键点是：只要替换公平性指标，整套框架就能适配不同偏置类型，而且不需要重新完整微调整个 LLM。
