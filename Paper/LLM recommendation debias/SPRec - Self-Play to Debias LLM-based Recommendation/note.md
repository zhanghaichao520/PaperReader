# SPRec

这篇论文研究 LLM 推荐在用 SFT 和 DPO 对齐用户偏好时产生的同质化问题。作者发现，尤其在 DPO 训练下，模型会越来越偏向少数高频物品或名字相似的物品，导致推荐结果过度集中。

论文提出 **SPRec**，核心思想是把模型自己的历史输出当成负样本做 **self-play**。每一轮训练先做一次 SFT，用离线交互数据里的正样本强化正确偏好；然后再做一次 DPO，把真实交互当正样本、把上一轮模型生成的结果当负样本。这样一来，模型越是偏向某些“过度推荐”的物品，这些物品后续越会被当成需要压制的负反馈，从而自适应地重加权 DPO 学习过程。整体上，SPRec 通过“SFT 保持准确性 + 自博弈生成负样本抑制偏置”来减轻推荐同质化。
